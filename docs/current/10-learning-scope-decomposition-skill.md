# 학습 범위 분할 스킬

## 개요

이 스킬은 캡스톤(학습의 최종 목표)에서 출발해 역추적으로 필요한 지식 단위들을 도출하고, 선수관계로 연결된 DAG를 만든다.

DAG는 처음부터 완성된 구조가 아니다. 역추적으로 만든 초안에서 시작해, 학습 진행 중 학습자의 편집과 리팩토링으로 점진적으로 개선된다. 여러 캡스톤을 달성하면서 DAG는 학습자의 개인 지식 그래프로 성장한다.

---

## 핵심 개념

### 캡스톤

학습의 거시적 목표. 다음 두 조건을 반드시 만족한다:

1. **이진 성공 기준**: "성공했는가, 아닌가"로 판단할 수 있어야 한다. "얼마나 잘 했는가"는 캡스톤의 판단 기준이 아니다. 그것은 학습자의 만족도 문제다.
2. **구체적 수행 요건**: canonical 문제(계산, 증명, 분석 등) 또는 프로젝트(구현, 설계 등) 형태로 주어진다.

캡스톤은 학습자가 직접 가져올 수도 있고, LLM이 분야에 맞는 canonical 문제를 제안할 수도 있다.

**캡스톤 예시:**
- "유수 정리를 사용하여 ∫₋∞^∞ 1/(x²+1) dx를 계산한다" — 계산 결과가 맞으면 성공
- "PID 제어기로 드론 호버링 시뮬레이터를 구현한다" — 목표 고도를 유지하면 성공
- "테트리스를 구현한다" — 게임이 동작하면 성공

### 지식 노드

DAG의 기본 단위. 각 노드는 다음을 가진다:

| 필드 | 내용 |
|---|---|
| 이름 | 개념·기술·절차명 |
| 개요 | 이 노드에서 학습할 내용 요약 (2~4문장) |
| 선수 노드 목록 | 이 노드를 학습하기 전에 완료해야 할 노드들 |
| 상태 | 대기중 / 학습중 / 완료 / 제외 |

캡스톤도 DAG의 노드다. 모든 필요한 지식 노드를 선수로 가지며 상태는 대기중 / 달성으로 구분된다.

### DAG

지식 노드들이 선수관계로 연결된 방향성 비순환 그래프.

- **뿌리 노드**: 선수가 없는 기초 단위
- **중간 노드**: 일부 노드의 선수이면서 다른 노드에 의존하는 단위
- **캡스톤 노드**: 선수만 있고 후속이 없는 최종 목표 (리프 노드)

DAG는 살아있는 문서다. 역추적으로 만든 초안에서 시작해, 학습 중 필요에 따라 편집하고 리팩토링한다.

## 프로세스

### 1단계: 캡스톤 확정

**학습자가 목표를 가져온 경우:**
학습자의 목표를 캡스톤으로 구체화한다. 이진 성공 기준이 불명확하면 명확하게 다듬는다.

**학습자가 분야만 제시한 경우:**
LLM이 그 분야에서 의미 있는 canonical 문제 2~3개를 제안한다. 각 제안에 대해 "이 문제를 풀 수 있으면 이 분야의 핵심을 이해한 것"이라는 이유를 함께 설명한다. 학습자가 하나를 선택하거나 조정한다.

### 2단계: DAG 초안 생성 (역추적)

캡스톤에서 출발해 "이것을 달성하려면 무엇을 알아야 하는가?"를 재귀적으로 물어 지식 노드들을 도출한다. 더 이상 선수가 없는 기초 단위에 도달하면 멈춘다.

- 모든 노드에 이름과 개요를 작성한다.
- 선수관계는 역추적 과정에서 자연스럽게 결정된다. 별도로 설정하지 않는다.
- 불확실한 영역은 웹 탐색을 수행하고 결과를 반영한다.
- 완성된 초안 전체를 한 번에 제시한다.

### 3단계: 초안 검토

학습자가 초안 DAG를 검토하고 각 노드에 대해 결정한다:

| 결정 | 의미 |
|---|---|
| 더 들어가기 | 이 노드를 더 세분화한다 |
| 여기서 멈추기 | 이 노드를 학습 단위로 확정한다 |
| 버리기 | 이 노드를 범위에서 제외한다 (이미 알거나 필요 없음) |

자연어로 직접 지시할 수도 있다: "A와 B를 하나로 합쳐줘", "C 노드를 더 구체적으로 풀어줘", "선수관계를 다시 검토해줘" 등.

### 4단계: 학습 진행

선수관계 순서대로 노드를 학습하고, 각 노드마다 지식 이해 테스트를 통과한다.

학습 중 DAG를 수정할 수 있다:

**노드 분해**: 단위가 너무 크다고 느껴지면 그 노드에 대해 역추적을 수행해 하위 노드들로 대체한다.

**직접 편집**: 노드 이름·개요·선수관계를 직접 수정한다.

**자연어 지시**: 변경 사항을 설명하면 LLM이 DAG를 업데이트한다.

수정이 잦아 DAG가 일관성을 잃으면 전반적 리팩토링을 요청할 수 있다. LLM이 현재 DAG 전체를 재검토해 더 일관된 구조로 재구성한 새 초안을 제시하고, 학습자가 승인하거나 조정한다.

### 5단계: 캡스톤 도전

모든 필수 노드를 완료한 후 캡스톤에 도전한다. 이진 기준으로 성공 여부를 판정한다.

---

## 다중 캡스톤 통합

새 캡스톤이 생기면:

**① 새 캡스톤 DAG를 별도로 생성한다**
기존 DAG와 무관하게 새 캡스톤에 대한 역추적을 수행해 독립적인 초안을 만든다.

**② 기존 DAG와 통합 리팩토링을 수행한다**
두 DAG를 비교해 다음 전략을 적용한다:

| 상황 | 처리 |
|---|---|
| 두 DAG에 동일한 노드 | 하나로 유지, 두 캡스톤이 공유 |
| 내용이 부분적으로 겹치는 노드 | 공통 부분과 캡스톤별 부분으로 분할 |
| 새 DAG에만 있는 노드 | 기존 DAG에 추가 |
| 구분이 불필요한 유사 노드 | 하나로 통합 |

통합 결과: 모든 캡스톤이 리프 노드인 단일 DAG.

## 시스템 프롬프트

```
당신은 학습 목표(캡스톤)에서 출발해 지식 DAG를 생성하고 관리하는 역할을 맡는다.

# 역할

학습자가 분야 또는 목표를 제시하면:
1. 캡스톤을 확정한다.
2. 캡스톤에서 역추적해 지식 노드 DAG 초안을 생성한다.
3. 학습자의 편집 지시와 리팩토링 요청에 응답해 DAG를 업데이트한다.
4. 새 캡스톤이 추가되면 기존 DAG와 통합 리팩토링을 수행한다.

# 캡스톤 규칙

캡스톤은 이진 성공 기준을 가져야 한다. "잘 했는가"가 아니라 "성공했는가"로 판단한다.
"얼마나 잘 했는가"는 학습자의 만족도 문제이며 캡스톤 판정에 포함하지 않는다.

학습자가 목표를 가져오면 그것을 캡스톤으로 다듬는다. 이진 기준이 불명확하면 명확하게 만든다.

학습자가 분야만 제시하면 canonical 문제 2~3개를 [캡스톤 제안 형식]으로 제안한다.
각 제안에 "이 문제를 풀면 이 분야의 무엇을 이해한 것인지" 이유를 함께 설명한다.

# 역추적 규칙

캡스톤에서 출발해 "이것을 달성하려면 무엇을 알아야 하는가?"를 재귀적으로 묻는다.
더 이상 선수가 없는 기초 단위에 도달하면 멈춘다.

모든 노드에 이름과 개요(2~4문장)를 작성한다.
선수관계는 역추적 과정에서 결정된다. 별도로 설정하지 않는다.
불확실한 영역은 웹 탐색을 수행하고 결과를 반영한다.
완성된 초안 전체를 [DAG 문서 형식]으로 한 번에 제시한다.

# 편집 및 리팩토링 규칙

학습자의 편집 지시를 처리한다:

노드 분해 요청: 해당 노드에 대해 역추적을 수행해 하위 노드들로 대체한다.
노드 병합 요청: 두 노드를 하나로 합치고 선수관계를 재정리한다.
선수관계 수정: 지시한 대로 변경한다.
전반적 리팩토링 요청: 현재 DAG 전체를 재검토해 더 일관된 구조로 재구성한 새 초안을 제시한다.
  학습자가 승인하거나 조정한다.

# 다중 캡스톤 통합 규칙

새 캡스톤이 추가되면:
1. 새 캡스톤에 대한 역추적 DAG를 독립적으로 생성한다.
2. 기존 DAG와 비교해 통합한다:
   - 동일 노드 → 유지 (공유)
   - 부분 겹침 → 분할
   - 새 노드 → 추가
   - 불필요한 구분 → 통합
3. 통합된 DAG를 제시하고 학습자가 검토한다.
4. 최종 DAG는 모든 캡스톤이 리프 노드인 구조가 된다.

# 응답 규칙

캡스톤 제안 시 [캡스톤 제안 형식]을 따른다.
DAG 출력 시 [DAG 문서 형식]을 따른다.
웹 탐색을 수행했다면 응답 상단에 간략히 명시한다.

# 입력 형식

학습 대상: [분야 또는 구체적 목표]
```

## 사용 예시

**입력:** `학습 대상: 복소해석학`

**1차 응답 (캡스톤 제안):**

```
복소해석학에서 다음 캡스톤을 제안합니다.

① 이상적분 계산
   문제: ∫₋∞^∞ 1/(x²+1) dx를 유수 정리를 사용하여 계산한다.
   성공 기준: π라는 정답에 도달하고 각 단계의 근거를 서술할 수 있다.
   이유: 유수 정리는 복소해석학의 핵심 결과다. 이 계산 하나에 복소수, 정칙함수,
         코시 적분 공식, 유수 개념이 모두 사용된다.

② 정칙성 판별
   문제: f(z) = z², g(z) = |z|에 대해 각각 코시-리만 방정식을 검증하고
         정칙성을 판단한다.
   성공 기준: 두 함수의 정칙성 여부를 올바르게 판단하고 근거를 서술할 수 있다.
   이유: 코시-리만 방정식은 복소 미분가능성의 정의다. 실해석학과의 결정적 차이가
         여기서 드러난다.

③ 조화 켤레 구성
   문제: u(x,y) = x²−y²가 조화함수임을 보이고 조화 켤레를 구성한다.
   성공 기준: 라플라스 방정식을 만족함을 보이고 조화 켤레를 올바르게 구성한다.
   이유: 조화함수는 정칙함수의 실수·허수 성분이며 물리학적 응용의 기초가 된다.

하나를 선택하거나 조정이 필요하면 말씀해주세요.
```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
캡스톤: 이상적분 계산
성공 기준: ∫₋∞^∞ 1/(x²+1) dx = π를 유수 정리로 계산하고 각 단계의 근거를 서술한다.
