# Change: Infinite Canvas DAG

Version: v2.1.0
Date: 2026-06-22
Status: Applied
Type: MINOR

---

## Why

v2.0.0에서는 DAG 시각화를 계층형 카드 리스트와 간단한 edge 표시 중심으로 정의했다. 이 방식은 구현 안정성은 높지만, DAG가 커질수록 지식 구조를 공간적으로 탐색하기 어렵다.

학습 DAG는 단순한 체크리스트가 아니라 살아있는 지식 지도다. 따라서 그래프의 전체 구조, 선수관계, 캡스톤까지의 경로, 현재 학습 가능한 노드를 공간적으로 파악할 수 있어야 한다.

---

## Before

- DAG 시각화는 좌측 패널 또는 카드 리스트 중심이었다.
- 현재 학습 가능한 노드와 상태 전이를 우선했다.
- 고급 그래프 시각화는 2차 또는 3차 구현으로 미뤘다.

---

## After

- DAG 시각화는 조작 패널과 분리된 무한캔버스에서 제공한다.
- 무한캔버스는 pan, zoom, 노드 선택, 노드 배치, edge 시각화, 그래프 탐색을 담당한다.
- Side Inspector와 Command Panel은 선택 노드 정보, LLM 명령, GraphPatch 검토, 테스트/학습 액션을 담당한다.
- 카드 리스트는 기본 UX가 아니라 모바일, 접근성, 렌더링 실패 대응용 fallback으로 유지한다.

## Domain Impact

도메인 모델의 의미론적 DAG 구조는 바꾸지 않는다.

추가되는 것은 `CanvasState`다.

```ts
type CanvasState = {
  viewport: { x: number; y: number; zoom: number };
  nodePositions: Record<NodeId, { x: number; y: number }>;
  selectedIds: string[];
  focusedNodeId?: string;
  layoutMode: 'auto' | 'manual' | 'mixed';
};
```

중요 규칙:

- 노드 드래그는 위치만 바꾼다.
- 노드 드래그는 선수관계를 바꾸지 않는다.
- edge 변경은 GraphPatch 또는 명시적 edge action으로만 수행한다.

---

## Non-Goals

이번 변경은 다음을 포함하지 않는다.

- 고급 자동 레이아웃
- 실시간 협업 그래프 편집
- 복잡한 그래프 애니메이션
- 다중 캡스톤 통합 UX
- 모바일 최적화

---

## Migration Notes

v2.0.0의 카드 리스트 기반 DAG UI는 삭제하지 않고 fallback으로 유지한다. MVP에서는 무한캔버스의 기능 범위를 pan/zoom/node select/basic layout으로 제한한다.
