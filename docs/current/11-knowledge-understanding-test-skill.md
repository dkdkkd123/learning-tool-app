# 지식 이해 테스트 생성 및 채점 스킬

## 개요

이 스킬은 지식 이해를 진단하는 테스트를 생성하고 채점한다.

핵심 원칙: **이해는 생산으로만 검증된다.** 외연을 만들고, 분류하고, 정의를 설계하고, 규칙을 적용한 결과로 측정한다.

테스트는 5가지 유형으로 구성된다.

- **유형 1** 내포 → 외연 생성
- **유형 2** 외연 분류
- **유형 3** 내포 판단
- **유형 4** 내포 디자인 (역방향: 외연 → 내포)
- **유형 5** 내포 적용

### 유형별 요약

| 유형 | 묻는 것 | 적합한 대상 |
|---|---|---|
| 1 | 정의를 만족하는 사례를 만들 수 있는가 | 분류적·정의적 개념 |
| 2 | 주어진 사례를 정의에 따라 분류할 수 있는가 | 분류적·정의적 개념 |
| 3 | 정의로부터 명제의 참·거짓을 논증할 수 있는가 | 분류적·정의적 개념 |
| 4 | 사례들로부터 정의·규칙을 역추론해 설계할 수 있는가 | 분류적·정의적 개념, 코드 구현 |
| 5 | 내포를 연역적으로 적용해 답을 도출할 수 있는가 | 절차적·연산적 개념 |

유형은 학습 대상의 성격에 따라 선택한다. 하나의 학습 대상에 여러 성격이 있다면 여러 유형을 함께 출제할 수 있다.

---

## 난이도 체계

각 유형의 난이도는 유형별로 정해진 **고유한 복잡도 손잡이**에 의해 결정된다.
문제 수를 늘리는 것은 난이도 조절이 아니다. 형식 제약 추가는 난이도 조절의 보조 수단이 될 수 있으나 주된 손잡이는 아니다.

| 유형 | 난이도 손잡이 |
|---|---|
| 1 (외연 생성) | 내포 조건의 수 + 조건 조합의 비전형성 |
| 2 (외연 분류) | 제시되는 내포의 복잡도(조건 수, 조건 간 상호작용) + 경계 사례의 교묘함 |
| 3 (내포 판단) | 추론 단계 수 + 반례 구성의 난이도 |
| 4 (내포 디자인) | 외연→내포 역추론의 추상화 도약 크기 |
| 5 (내포 적용) | 적용 단계 수 + 대상의 규모·복잡도 |

## 사용 방법

### 1단계: 시험지 생성

다음 형식으로 입력한다:

```
학습 대상: [개념명 또는 텍스트(발췌·챕터 등)]
난이도: [초급 / 중급 / 고급]
유형별 문제 수: 유형1: n개, 유형2: n개, 유형3: n개, 유형4: n개, 유형5: n개
```

학습 대상은 경계가 명확하고 잘 알려진 개념명일 수도 있다.
특정 전공서적의 한 챕터, 법조문 발췌처럼 LLM이 사전에 알지 못하거나 경계가 불분명한 내용이라면 본문 자체를 입력해야 한다. 어느 쪽이든 LLM은 그로부터 내포(정의·규칙·절차)를 추출해 출제한다.

LLM은 [시험지 형식]에 따라 시험지를 생성하고,
[답안지 형식]에 따라 답안지를 별도 섹션에 생성한다.
(답안지는 학습자가 답안을 작성한 후에 공개한다.)

### 2단계: 채점

학습자가 답안을 제출하면 [채점 결과 형식]에 따라 채점한다.

## 테스트 유형 정의

### 유형 1: 내포 → 외연 생성

**목적:** 정의를 읽고 그것을 만족하는 구체적 사례를 직접 구성할 수 있는가.

**난이도 구성 기준:**

| 난이도 | 내포 조건 수 | 사례 요건 |
|---|---|---|
| 초급 | 1~2개 | 교과서적·전형적 사례로 충분 |
| 중급 | 2~3개 (일부 조건이 서로 제약) | 전형적 사례 이외의 창의적 구성 필요 |
| 고급 | 조건 다수 또는 서로 교차·충돌하는 조건들 | 조건들을 동시에 만족하는 비전형적 사례 요구 |

**출제 규칙:**
1. 내포(정의)를 명시적으로 제시한다.
2. 출제자는 반드시 예시 답안을 최소 1개 마련해둔다.
3. 예시 답안은 내포의 각 조건에 구절 대응하는 풀이를 포함해야 한다.
4. 출제자가 예시 답안을 마련할 수 없다면 문제 출제 자체를 재검토한다.

**학습자 수행 요건:**
- 내포를 만족하는 외연(사례) 1개를 직접 만든다.
- 만든 사례가 내포의 각 조건을 어떻게 만족하는지 조건 구절마다 일대일 대응하여 서술한다.
- 사고의 흐름이 드러나도록 과정을 명시적으로 작성한다.

**채점 기준:**
- 외연이 내포의 모든 조건을 만족하는가 (O/X)
- 조건 구절마다 대응 설명이 명확한가
- 풀이 과정에서 오개념이 없는가

---

### 유형 2: 외연 분류

**목적:** 주어진 사례들을 내포에 근거하여 정확히 분류할 수 있는가.

**난이도 구성 기준:**

| 난이도 | 제시되는 내포 | 경계 사례 구성 |
|---|---|---|
| 초급 | 단일 조건 | 없음 또는 전형적 O/X만 |
| 중급 | 복수 조건 (AND/OR 관계 명확) | 경계 사례 1~2개 (조건 중 일부만 만족) |
| 고급 | 복수 조건 + 조건 간 상호작용 | 직관을 배신하는 경계 사례 다수 (한 조건은 만족하지만 다른 조건에서 탈락하는 사례, 또는 그 반대) |

**출제 규칙:**
1. 내포(정의)를 반드시 명시적으로 제시한다. (암기 측정이 아니라 조건 판단 능력 측정이 목적)
2. 분류할 외연 목록을 제시한다. 다음을 포함한다:
   - 명백하게 O인 사례
   - 명백하게 X인 사례
   - 혼동하기 쉬운 경계 사례 (반드시 포함)
3. 출제자는 각 외연의 정답과 내포 조건에 근거한 판단 근거를 답안지에 마련해둔다.

**학습자 수행 요건:**
- 각 외연에 O/X를 표시한다.
- O/X 판단의 근거를 내포의 조건에 근거하여 서술한다.
- 사고의 흐름이 드러나도록 과정을 명시적으로 작성한다.

**채점 기준:**
- O/X 정답 여부
- 근거가 내포 조건에 올바르게 근거하는가
- 정답이더라도 근거가 틀렸다면 오개념으로 기록한다

---

### 유형 3: 내포 판단

**목적:** 개념의 내포에 관한 명제가 참인지 거짓인지 논리적으로 판단할 수 있는가.

**난이도 구성 기준:**

| 난이도 | 추론 구조 |
|---|---|
| 초급 | 정의를 1회 직접 적용하여 참/거짓 판단 가능 |
| 중급 | 2단계 이상 추론, 또는 정의로부터 새로운 성질을 도출해야 판단 가능 |
| 고급 | 다단계 추론 또는 반례를 직접 구성해야 함. 참처럼 보이는 거짓 명제 포함 |

**출제 규칙:**
1. 명제를 제시한다.
2. 출제자는 답안지에 해당 명제의 논리적 증명을 작성한다.
   - 증명은 직관이나 직감에 의존하지 않는다.
   - 정의(내포)에서 출발하여 논리적으로 도달하는 과정을 보인다.
   - 반례 제시 또는 연역적 논증 중 적절한 방식을 선택한다.

**학습자 수행 요건:**
- O/X를 표시한다.
- 판단의 근거를 논리적으로 서술한다. (직감이 아닌 정의에 근거)
- 사고의 흐름이 드러나도록 과정을 명시적으로 작성한다.

**채점 기준:**
- O/X 정답 여부
- 근거가 논리적으로 타당한가 (정의에서 출발하는가)
- 정답이더라도 근거가 직감에만 의존하면 불완전한 이해로 기록한다

---

### 유형 4: 내포 디자인

**목적:** 제시된 외연을 모두 만족하는 내포(정의·규칙)를 직접 설계할 수 있는가.
코드에서 테스트 케이스를 먼저 쓰고 그것을 통과하는 구현을 만드는 것과 동일한 구조이며, 코드 구현 문제도 이 유형에 속한다.

**난이도 구성 기준:**

유형 4의 난이도는 **외연에서 내포를 역추론할 때 필요한 추상화 도약의 크기**로 결정된다.
추상화 도약의 크기는 다음 세 요소로 결정된다:

- **패턴의 시각적 명확성**: 외연을 보자마자 공통 속성이 드러나는가, 계산·분석을 해야 보이는가
- **가설 오염 여부**: 그럴듯한 첫 가설을 깨는 반례(O이지만 직관적으로 X처럼 보이는 사례, 또는 그 반대)가 포함되어 있는가
- **내포의 비자명성**: 올바른 내포가 학습자의 직관적 접근 방향에 있는가, 아닌가

| 난이도 | 추상화 도약 | 사례 구성 원칙 |
|---|---|---|
| 초급 | 최소 | O 사례들의 공통 속성이 시각적으로 즉각 드러남. 내포를 언어화하는 것이 주된 과제. |
| 중급 | 보통 | 패턴을 찾으려면 계산이나 추가 확인이 필요. 그럴듯한 첫 가설을 깨는 반례 1~2개 포함. |
| 고급 | 최대 | 직관에 반하는 O 사례 포함(처음 가설이 깨짐). 여러 가설을 수정해야 올바른 내포에 도달. 내포 자체가 비자명한 구조. |

**주의**: 유형 4의 주된 난이도 손잡이는 추상화 도약의 크기다. 형식적 제약(예: "조건제시법으로 작성") 추가는 보조적인 난이도 조절 수단이 될 수 있으나, 이것만으로 난이도를 올리려 하면 추상화 도약 크기를 건드리지 않은 채 표면만 어려워진다.

**출제 규칙:**
1. 주제(개념명)를 제시하지 않거나, 힌트가 되지 않는 중립적 명칭으로 제시한다.
2. 외연 목록을 O/X와 함께 제시한다. 다음을 포함한다:
   - 명백한 O 사례
   - 명백한 X 사례
   - 학습자의 첫 번째 가설을 깨는 반례 (난이도에 따라 수와 교묘함을 조정)
3. 외연 충족 외에 추가로 지켜야 할 형식 제약이 있다면 문제에 명시한다. 불필요한 제약은 추가하지 않는다.
4. 외연이 코드로 검증되는 경우(예: 함수 구현), 외연 목록을 실제 테스트 케이스로 삼아 학습자의 구현을 실행·검증할 수 있는 코드를 답안지에 마련해둔다.
5. 출제자는 답안지에 다음을 마련해둔다:
   - 모범 정의(또는 구현)
   - 각 외연이 모범 정의에 의해 어떻게 분류되는지 설명
   - 정의가 만족해야 할 핵심 조건 목록

**학습자 수행 요건:**
- 제시된 모든 외연과 추가 제약조건을 만족하는 정의(또는 구현)를 작성한다.
- 자신의 정의가 각 외연을 어떻게 분류하는지 대응하여 보인다.
- 사고의 흐름(어떻게 이 정의에 도달했는가)을 명시적으로 작성한다.

**채점 기준:**
- 학습자의 정의(구현)가 제시된 모든 외연을 올바르게 분류하는가 (코드라면 검증 코드로 테스트 케이스를 모두 통과하는가)
- 추가 제약조건이 있었다면 그것도 만족하는가
- 정의가 제시되지 않은 임의의 사례에도 적용 가능한가 (선택적 추가 검증)
- 모든 조건을 만족하지만 직관적으로 틀린 정의라면 → 정의 재설계 피드백
- 사고 과정에서 드러나는 오개념 확인

### 유형 5: 내포 적용

**목적:** 문제 상황과 내포가 주어졌을 때, 내포를 연역적으로 적용하여 궁금한 대상에 대한 답을 도출할 수 있는가.

**난이도 구성 기준:**

| 난이도 | 적용 구조 |
|---|---|
| 초급 | 단일 내포를 1~2회 단순 적용. 소규모·단순 대상. |
| 중급 | 2~3단계 연쇄 적용, 또는 여러 내포 중 적용 가능한 것을 식별 필요. |
| 고급 | 다단계 연쇄 + 규모 확장(더 큰 대상, 더 복잡한 조건). 예외·경계 케이스 처리 포함. |

**출제 규칙:**
1. 문제 상황(초기 조건), 적용할 내포(규칙·정의·성질), 궁금한 대상(질문)을 명시적으로 제시한다. 적용할 내포를 학습자가 스스로 시험지 외부에서 찾아야 하는 문제는 출제하지 않는다.
2. (선택) 대상 개념과 개념적으로 연관되어 헷갈릴 법한 미끼 내포를 1개 이상 함께 제시할 수 있다. 풀이와 무관한 미끼는 변별력이 없으므로 피한다.
3. 내포를 한 번만 기계적으로 대입하면 풀리는 문제는 지양한다. 두 단계 이상 연쇄적으로 적용해야 풀리도록 설계하되, 학습자가 감당할 수 있는 범위로 제한한다.
4. 결과가 유일하지 않을 수 있는 문제라면 결과가 만족해야 할 불변조건을 답안지에 명시한다.
5. (권장) 경계·예외 상황을 하나 이상 포함한다.
6. 출제자는 답안지에 단계별 모범 풀이를 마련하고, 각 단계가 내포의 어떤 조건에서 비롯되는지 대응시킨다. 모범 풀이를 마련할 수 없다면 문제를 재검토한다.

**학습자 수행 요건:**
- 내포가 여러 개 제시된 경우, 실제로 적용되는 것을 식별한다.
- 초기 조건에 내포를 연역적으로 적용해 궁금한 대상에 대한 답을 도출한다. 필요하면 여러 단계에 걸쳐 적용한다.
- 풀이 과정을 단계별로 작성하며, 각 단계가 어느 조건에 근거하는지 밝힌다.

**채점 기준:**
- 결과가 정답(또는 불변조건)을 만족하는가
- 풀이 과정이 내포를 단계마다 올바르게 따랐는가 (우연한 정답과 구분)
- 경계·예외 상황이 포함된 문제라면 처리의 정오를 별도로 기록한다

## 시스템 프롬프트

이 문서를 사용하는 LLM에게 전달할 프롬프트:

```
당신은 지식 이해 테스트를 생성하고 채점하는 역할을 맡는다.

# 역할

사용자가 학습 대상(개념명 또는 텍스트), 난이도, 유형별 문제 수를 입력하면:
1. 위 [시험지 형식]에 따라 시험지를 생성한다.
2. 위 [답안지 형식]에 따라 답안지를 생성한다. (답안지는 "답안지 — 비공개"라고 표시하고 별도 섹션에 위치시킨다.)

사용자가 학습자의 답안을 제출하면:
3. 위 [채점 결과 형식]에 따라 채점 결과를 생성한다.

# 핵심 원칙

1. 단순 암기를 측정하지 않는다. 유형 1·2·3·4는 외연을 생성·분류·설계하는 능력을 측정하고, 유형 5는 내포를 연역적으로 적용하는 능력을 측정한다.

2. 난이도 적용 규칙:
   - 난이도는 각 유형의 [난이도 구성 기준] 표에 따라 문제 인스턴스에 적용한다.
   - 문제 수를 늘리는 것으로 난이도를 올리지 않는다. 형식 제약 추가는 보조 수단이 될 수 있으나 주된 손잡이가 아니다.
   - 유형 1: 내포 조건 수와 사례 비전형성을 난이도에 따라 조정한다.
   - 유형 2: 제시하는 내포의 복잡도(조건 수, 상호작용)와 경계 사례의 교묘함을 조정한다.
   - 유형 3: 추론 단계 수와 반례 구성 여부를 조정한다.
   - 유형 4: 외연→내포 역추론의 추상화 도약 크기를 조정한다. 패턴의 시각적 명확성, 가설 오염 사례 수, 내포의 비자명성을 난이도에 따라 설계한다.
   - 유형 5: 적용 단계 수와 대상 규모를 조정한다.

3. 출제 시 반드시 지켜야 할 규칙:
   - 유형 4 출제 시 주제명이 내포를 직접 노출하지 않도록 중립적으로 제시하거나 생략한다.
   - 유형 1: 예시 답안 최소 1개와 조건 구절 대응 풀이를 답안지에 마련한다.
   - 유형 2: 내포를 시험지에 명시적으로 제시한다. 경계 사례를 반드시 포함한다.
   - 유형 3: 답안지에 정의에서 출발하는 논리적 증명을 작성한다. 직감에 의존하지 않는다.
   - 유형 4: 외연 목록에 학습자의 첫 가설을 깨는 반례를 포함한다. 추가 제약조건이 있다면 명시한다. 코드 문제라면 외연을 테스트 케이스로 삼는 검증 코드를 답안지에 마련한다.
   - 유형 5: 문제 상황·내포·궁금한 대상을 명시적으로 제시한다. 내포를 두 단계 이상 적용해야 풀리도록 설계하되 학습자가 감당할 수 있는 범위를 넘지 않게 한다.

4. 채점 시 반드시 지켜야 할 규칙:
   - 객관식 문제(유형 2·3): 사전에 정해진 정답과 학습자 답을 비교한다.
   - 서술형 문제(유형 1·4): 학습자 답이 내포의 조건들(및 제약조건)을 만족하는지 확인한다.
   - 유형 5: 결과가 정답(불변조건)을 만족하는지 확인하고, 풀이 과정이 내포를 단계마다 올바르게 따랐는지 본다. 결과가 맞아도 과정이 잘못됐다면(우연한 정답 등) 오개념으로 기록한다.
   - 모든 유형: 학습자의 풀이 과정을 읽고 오개념이 없는지 확인한 후 피드백을 작성한다.
   - 정답이더라도 근거나 논증이 잘못되었다면 오개념으로 기록하고 피드백을 제공한다.
   - 종합 피드백에 다음 시험 권장 난이도를 명시한다. (마지막 시도 통과 → 다음 난이도 승급 / 미통과 → 동일 난이도 재시험)

5. 학습이 목적임을 잊지 않는다. 채점 결과와 피드백은 정확히 어디에 오개념이 있는지 위치를 특정하도록 작성한다.

# 입력 형식

학습 대상: [개념명 또는 텍스트(발췌·챕터 등)]
난이도: [초급 / 중급 / 고급]
유형별 문제 수: 유형1: n개, 유형2: n개, 유형3: n개, 유형4: n개, 유형5: n개
```

## 사용 예시

**입력:**
```
학습 대상: 단위 쿼터니언
난이도: 초급
유형별 문제 수: 유형1: 1개, 유형2: 1개, 유형3: 1개, 유형4: 1개, 유형5: 1개
```

**시험지 5문항, 각 유형이 묻는 것:**
- 유형 1: 단위 쿼터니언의 정의를 만족하는 사례를 직접 구성 (초급: 조건 1~2개)
- 유형 2: 주어진 사원수들이 단위 쿼터니언인지 분류 (초급: 단일 조건, 전형적 O/X)
- 유형 3: 단위 쿼터니언에 관한 명제의 참·거짓을 논증 (초급: 정의 1회 적용)
- 유형 4: 제시된 사원수 사례들로부터 해당 집합의 정의를 역추론 (초급: 패턴이 즉각 드러나는 외연 구성)
- 유형 5: 회전 규칙을 구체적 벡터에 적용해 회전 결과를 도출 (초급: 단일 적용)

답안지는 각 유형의 모범 답안과 채점 기준을 비공개 섹션에 담아 함께 생성된다.
학습자 제출 후에는 [채점 결과 형식]에 따라 문항별 정오, 오개념 위치, 피드백과 종합 피드백이 출력된다.
종합 피드백 말미에 다음 시험 권장 난이도(동일 난이도 재시험 또는 중급 승급)가 명시된다.
